

## Генерация изображений и товаров для К.АЯ

### Что будет сделано

**1. Создание storage-бакета для изображений**
- Создаём публичный бакет `product-images` для хранения фото товаров и категорий

**2. Генерация изображений через Lovable AI (Nano Banana 2)**
- Для каждой из 4 категорий — по 1 атмосферному фото для главной страницы (зигзаг-секции)
- Для каждого товара — по 1 фото продукта на нейтральном кожаном фоне
- Используем модель `google/gemini-3.1-flash-image-preview` для качественной генерации
- Все изображения загружаются в бакет и URL сохраняются в БД

**3. Создание товаров (3-4 в каждой категории, ~14 штук)**

| Категория | Товары |
|-----------|--------|
| Сумки | Шоппер «Маршрут», Кросс-боди «Прогулка», Клатч «Вечер» |
| Докхолдеры | Докхолдер «Классика», Докхолдер «Компакт», Обложка на паспорт «Путь» |
| Ремни | Ремень «Основа», Ремень «Плетёный», Ремень «Минимал» |
| Кошельки | Портмоне «Надёжный», Кошелёк «Компакт», Картхолдер «Лёгкий», Монетница «Круг» |

Каждый товар: название, slug, описание, цена, категория, сгенерированное фото.

**4. Обновление категорий** — добавляем `image_url` к каждой категории

**5. Обновление главной страницы** — зигзаг-секции будут показывать реальные фото из БД вместо плейсхолдеров

### Технические детали

- Edge-функция `generate-images` для генерации изображений через Lovable AI и загрузки в Storage
- Миграция: создание storage-бакета `product-images` с публичным доступом
- INSERT товаров через insert tool после генерации изображений
- Обновление `Index.tsx`: загрузка категорий из БД с фото
- Все товары сразу видны в админке для редактирования
