

## Проблема

Сейчас при заказе через бота происходит двойное оформление:
1. Бот автоматически создаёт заказ в БД (chat-assistant, строки 181-206) с `delivery_method: "pickup"` без вопроса
2. Кнопка "Перейти в корзину" добавляет товары в корзину, и пользователь оформляет **второй** заказ через форму Cart — отсюда дубликат в админке

### Решение

Убрать автоматическое создание заказа из chat-assistant. Бот должен только формировать блок ORDER для отображения в чате, а реальный заказ создаётся один раз — через корзину, где пользователь выбирает доставку/самовывоз и указывает адрес.

### Изменения

**1. `supabase/functions/chat-assistant/index.ts`**
- Удалить блок автоматического создания заказа (строки 154-244: парсинг ORDER, insert в orders/order_items, Telegram-уведомление)
- Оставить только сохранение сообщения ассистента в chat_messages
- В промпте добавить инструкцию: после ORDER сказать клиенту перейти в корзину для выбора доставки

**2. `src/components/ChatWidget.tsx`**
- OrderBlock остаётся как есть — добавляет товары в корзину и ведёт на страницу Cart
- Без изменений

**3. Промпт (в chat-assistant)**
- Добавить текст: "После блока ORDER скажи клиенту нажать кнопку 'Перейти в корзину', где можно выбрать самовывоз или доставку и указать адрес"

Итог: один заказ, одно уведомление в Telegram (из Cart.tsx), с правильным способом доставки.
