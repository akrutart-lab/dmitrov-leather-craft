

## Исправление формы заказа в корзине

### Проблемы
1. **Радио-кнопки доставки** — переключение между "Самовывоз" и "Доставка" может не работать из-за того, что `<label>` не всегда корректно пробрасывает клик на скрытый `<input>` (`sr-only`). Решение: добавить явный `onClick` на `<label>`.
2. **Нет поля адреса** — при выборе "Доставка" нет поля для ввода адреса.
3. **Ошибка отправки** — возможна скрытая ошибка при вставке в БД (например, несовпадение типов). Добавим лучший вывод ошибок.

### Изменения

**Файл: `src/pages/Cart.tsx`**

1. Заменить радио-кнопки на обычные `<button>` с явным `onClick` для переключения способа доставки — это надёжнее, чем скрытый `<input type="radio">`.

2. Добавить поле "Адрес доставки", которое появляется только при выборе `delivery`.

3. Добавить `address` в состояние формы и передавать его в `customer_comment` (или отдельным полем, если нужна миграция).

4. Улучшить обработку ошибок — логировать ошибку в консоль для отладки.

**Миграция БД** — добавить поле `delivery_address` в таблицу `orders` (опциональное текстовое поле).

### Результат
- Переключение "Самовывоз / Доставка" работает надёжно
- При выборе "Доставка" появляется поле для адреса
- Заявка корректно отправляется в БД
