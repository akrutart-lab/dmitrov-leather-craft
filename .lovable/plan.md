

## Карточки товаров в чате + оформление заявки ботом

### Что будет сделано

**1. Показ карточек товаров в чате**

ИИ уже знает каталог, но не может показать фото. Решение — добавить в контекст ИИ URL изображений и slug, а в системный промпт инструкцию использовать специальный markdown-формат для карточек товара. На клиенте — парсить эти блоки и рендерить как мини-карточки с фото, ценой и кнопкой "В корзину".

- Edge-функция: добавить `image_url`, `id` в выборку products, передать в каталог формат: `[PRODUCT:id:slug:name:price:image_url]`
- Системный промпт: инструкция использовать `[PRODUCT:...]` когда показываешь товар покупателю
- ChatWidget: кастомный рендер — при встрече `[PRODUCT:...]` в тексте показывать карточку с изображением, названием, ценой и кнопкой "Добавить в корзину"

**2. Оформление заявки из чата**

Когда покупатель подтверждает покупку, бот формирует заявку. Решение — edge-функция после получения ответа ИИ проверяет наличие специального маркера `[ORDER_CONFIRM]` в ответе. Если маркер есть — создаёт заказ из товаров, обсуждённых в чате.

Альтернативный (более надёжный) подход: бот выводит кнопку "Оформить заявку" в чат, при нажатии клиент отправляет команду `/order` с товарами, edge-функция создаёт заказ.

- Системный промпт: когда покупатель готов, вывести блок `[ORDER:id1:qty1,id2:qty2]` с итогом
- ChatWidget: парсить `[ORDER:...]`, показывать сводку заказа с кнопкой "Подтвердить заявку"
- При нажатии: вызов edge-функции или прямая вставка в orders/order_items + добавление в корзину + переход на /cart, или прямое создание заказа из чата

### Реализация

**Edge-функция `chat-assistant`:**
- Расширить SELECT products: добавить `id, image_url`
- Изменить формат каталога: `[PRODUCT:id|name|price|image_url|slug]`
- Обновить системный промпт: "Когда показываешь товар — используй формат `[PRODUCT:id|name|price|image_url|slug]`. Когда покупатель подтвердил — используй `[ORDER:id1:qty1,id2:qty2|delivery_method|address]` и скажи что заявка оформлена"
- После стриминга: если в fullContent есть `[ORDER:...]` — парсить и создавать order + order_items в БД, используя customer_name/phone из chat_session

**ChatWidget.tsx:**
- Новый тип сообщения с поддержкой embedded карточек
- Кастомный рендер markdown: заменять `[PRODUCT:...]` на мини-карточку (фото, название, цена, кнопка "В корзину")
- Заменять `[ORDER:...]` на блок подтверждения заказа с итогом
- Кнопка "В корзину" вызывает `addToCart()` из `@/lib/cart`
- При блоке ORDER — показать сводку и кнопку "Перейти в корзину" (ссылка на /cart)

**Файлы:**
- `supabase/functions/chat-assistant/index.ts` — расширить каталог + промпт + auto-order
- `src/components/ChatWidget.tsx` — карточки товаров + блок заказа

### Результат
- Бот показывает товары с фото и ценами прямо в чате
- Покупатель может добавить товар в корзину из чата
- Бот может оформить заявку когда покупатель подтверждает покупку — заказ создаётся в БД автоматически
