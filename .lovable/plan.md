

## Полноценная админ-панель К.АЯ

Сейчас админка минимальна: список заказов со сменой статуса и простая форма редактирования товаров с вводом URL изображения вручную. Нужно сделать полноценную панель управления магазином.

### Что будет сделано

**1. Заказы — улучшенный вид**
- Фильтры по статусу (Все / Новые / В работе / Выполненные)
- Поле комментария администратора к заказу (миграция: добавить `admin_comment` в таблицу `orders`)
- Счётчик новых заказов на вкладке

**2. Товары — полное управление**
- Загрузка фото с компьютера (файл → Storage бакет `product-images`) вместо ввода URL
- Превью загруженного фото в форме редактирования
- Поддержка нескольких фото (галерея) — загрузка в массив `images`
- Автогенерация slug из названия (транслитерация)
- Drag-and-drop зона для загрузки фото

**3. Категории — новая вкладка**
- CRUD категорий: создание, редактирование, удаление
- Загрузка фото категории с компьютера
- Изменение порядка сортировки

**4. UI/UX улучшения**
- Три вкладки: Заказы / Товары / Категории
- Адаптивная вёрстка для мобильных
- Подтверждение при удалении через диалог вместо `confirm()`

### Технические детали

**Файлы:**
- `src/pages/Admin.tsx` — рефакторинг: разбить на компоненты, добавить вкладку категорий, загрузку файлов
- Новые компоненты: `src/components/admin/OrdersTab.tsx`, `src/components/admin/ProductsTab.tsx`, `src/components/admin/CategoriesTab.tsx`, `src/components/admin/ImageUpload.tsx`

**Миграция БД:**
- Добавить `admin_comment TEXT` в таблицу `orders`

**Storage:**
- Используется существующий бакет `product-images` — загрузка через `supabase.storage.from('product-images').upload()`

**Логика загрузки фото:**
- Пользователь выбирает файл → файл загружается в Storage → получаем публичный URL → сохраняем в `image_url` товара/категории
